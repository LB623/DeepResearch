# Prompt Quality Guards 固定集 E2E A/B 评测

## 2026-06-18 第二轮修复进展

第一轮失败后完成了以下确定性修复：

- Writer 修订时显式传入上一版草稿，避免“要求修改、实际从零重写”
- polish 输出增加长度、章节和 URL 白名单校验，不完整时回退完整 draft
- draft 输出对照 outline 校验章节，缺失时触发一次定向补全
- 最后一次 critic 反馈传入 polish，继续处理遗留 critical/major 问题
- Writer critic 改用 `reflection_model`，而非 `answer_model`
- Eval 将题目查询数/循环数显式写入研究 state，修复子图未继承配置的问题
- Eval 使用持久 asyncio runner，避免跨题关闭 event loop 时的 httpx 清理异常

## 当前结论

`PROMPT_QUALITY_GUARDS=1` 已将固定集幻觉率从 `60%` 降至 `0%`，事实准确性和逻辑结构也高于 baseline；但覆盖度和时效性回退使平均 E2E 从 `4.36` 降至 `4.24`。当前只能表述为“降低幻觉并改善事实约束”，不能表述为“提升整体回答质量”。

## 失败分析

1. **第 5 题终稿截断**：optimized 报告停在“4.1 AGENTS.md 的编写规范与结构模板”的开头，字符数仅 `5964`，未输出模板正文、风险分析和结论。Judge 因此将信息覆盖评为 `3/5`，并指出报告不完整。
2. **事实约束仍未闭环**：第 3 题把 2026-06-17 的事件写成 2025-06-17；第 4 题仍产生无材料支撑的 `MATCH 法案`，说明“只能使用 Summaries”尚未被可靠执行或校验。
3. **引用护栏只约束 URL，不保证来源权威性**：optimized 仍大量使用 CSDN、知乎、百家号和搜狐等二手来源，引用质量均值反而下降 `0.2`。
4. **更长输出不等于更高质量**：第 2 题 optimized 报告从 `8506` 字增长到 `14069` 字，但总体分从 `4.8` 降至 `4.2`，说明新增内容带来了范围漂移和低价值扩写。
5. **单次 Judge 存在随机性**：本结果足以判定“当前配置未显示稳定提升”，但不能仅凭一轮 A/B 精确估计真实效应量。

## 下一轮最小改动

1. 在 Writer polish 后增加结构完整性校验：终稿必须覆盖大纲的全部一级/二级标题，并以结论章节正常结束；不满足时回退到完整 draft，而不是接受截断 polish。
2. 将事实护栏从纯提示词升级为确定性校验：最终报告中的年份、数字、专名和 URL 必须能映射回 Summaries，无法映射的句子进入修订列表。
3. 对来源设置等级并在查询/摘要阶段优先官方、监管机构、公司公告和权威媒体，不能只验证 URL 是否存在。
4. 修复后在同一 5 题固定集上至少运行 3 个重复样本，报告均值、标准差、逐题胜率和幻觉率；不要挑选单题重跑后替换失败结果。

## 验收门槛建议

- 平均 E2E 不低于 baseline，且目标提升至少 `+0.2`
- 幻觉题数从 `3/5` 降至不高于 `1/5`
- 任一题不得出现终稿截断或章节缺失
- 五维均值不得有超过 `0.2` 的回退
